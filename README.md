# 设计模式项目简介

## 项目概述
本项目是一个全面的设计模式学习与实践项目，包含了23种经典GoF设计模式以及其他常用设计模式的Java实现。每个设计模式都有相应的示例代码和测试用例，便于学习和理解各种设计模式的应用场景和实现方法。

## 项目结构
项目按照不同的设计模式组织在各自的目录中，主要包括：

### 创建型模式 (Creational Patterns)
- **单例模式 (单例)** - 提供了多种单例实现方式（饿汉式、懒汉式、枚举等）
- **原型模式 (原型)** - 展示对象克隆机制
- **工厂模式 (工厂)** - 包含简单工厂、工厂方法和抽象工厂
- **建造者模式 (建造者)** - 分离构建复杂对象的过程与表示

### 结构型模式 (Structural Patterns)
- **适配器模式 (适配器)** - 类适配器、对象适配器和接口适配器
- **桥接模式 (桥接)** - 将抽象部分与实现部分分离
- **装饰者模式 (装饰者)** - 动态地给对象添加功能
- **组合模式 (组合)** - 将对象组合成树形结构
- **外观模式 (外观)** - 为子系统提供统一接口
- **享元模式 (享元)** - 运用共享技术有效地支持大量细粒度的对象
- **代理模式 (代理)** - 静态代理和动态代理（JDK和CGLIB）

### 行为型模式 (Behavioral Patterns)
- **策略模式 (策略)** - 定义算法族，分别封装起来
- **命令模式 (命令)** - 将请求封装成对象
- **责任链模式 (责任链)** - 请求的链式处理
- **模板方法模式 (模版)** - 定义算法框架
- **观察者模式 (观察者)** - 对象间的一对多依赖关系
- **中介者模式 (中介者)** - 对象间的交互封装
- **备忘录模式 (备忘录)** - 在不破坏封装性的前提下保存状态
- **解释器模式 (解释器)** - 给定语言的文法定义
- **状态模式 (状态)** - 对象的行为基于其状态而改变
- **访问者模式 (访问者)** - 操作与结构分离
- **迭代器模式 (迭代器)** - 提供遍历集合元素的方法

## 技术栈
- Java 8+
- Maven 3.6+
- CGLIB 3.3.0（用于代理模式演示）

## 项目特点
1. **完整覆盖** - 实现了所有经典的GoF设计模式
2. **易于理解** - 每个模式都有清晰的示例代码和测试用例
3. **实用性强** - 代码结构清晰，可以直接作为参考使用
4. **持续更新** - 项目会不断补充和完善新的设计模式示例

## 使用说明
每个设计模式目录中都包含：
- 核心实现类
- 测试类（Test.java）
- 相关接口和辅助类

运行测试类可以直观地看到每种设计模式的工作原理和效果。

## 学习建议
1. 先了解每种设计模式的概念和适用场景
2. 阅读代码实现，理解其内部机制
3. 运行测试用例，观察实际效果
4. 尝试修改代码，加深理解
